## üí° Problem-Solving Tips

### Step 1: Read Carefully
- Understand what language family is involved
- Note the type of data given (words, sentences, paradigms)
- Identify what the questions are asking

### Step 2: Look for Patterns
- Group similar forms together
- Look for repeated elements
- Compare related words/sentences
- Notice what changes and what stays the same

### Step 3: Form Hypotheses
- Make educated guesses about rules
- Test your hypothesis against all data
- Refine when you find exceptions

### Step 4: Check Your Work
- Verify your answers work for ALL examples
- Make sure translations make semantic sense
- Double-check any rules you've formulated

### Step 5: Review Solutions
- Read solutions even if you got it right
- Learn alternative approaches
- Understand linguistic terminology used

## ‚ùì FAQ

**Q: Which problems should I do first?**
A: Start with Breakthrough Workout, then Model Problems 1-5.

**Q: How do I know if a problem is right for my level?**
A: If you can solve it in 20-30 minutes with some thinking, it's perfect. Too easy = <10 min. Too hard = can't make any progress in 30 min.

**Q: What if I get stuck?**
A: Spend 5-10 more minutes, then check the solution. Learn from it and try a similar problem.

**Q: Should I time myself?**
A: For practice, no. For competition prep, yes (after you've solved it untimed first).

**Q: Can I use resources while solving?**
A: For learning, yes! For competition simulation, no. Practice both ways.

**Q: How many problems should I do per week?**
A: Beginners: 2-3. Intermediate: 5-7. Advanced: 10-15.
